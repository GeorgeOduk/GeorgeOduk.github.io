Global Layoffs Data Cleaning

A MySQL data cleaning project focused on improving data quality, standardising inconsistent values and preparing a raw global layoffs dataset for analysis.

Project Summary

This project demonstrates a structured MySQL data cleaning workflow applied to a raw global layoffs dataset. The aim was to improve the reliability, consistency and usability of the data before future exploratory analysis or reporting.

The cleaning process involved creating staging tables, identifying and removing duplicate records, standardising text fields, converting date values into a consistent format, handling null and blank values, filling missing industry values where possible and removing records without usable layoff information.

The project highlights an important foundation of analytics: raw data often needs to be cleaned and standardised before meaningful insights can be produced.

Tools Used

  • MySQL for data cleaning and transformation
  • MySQL Workbench for writing and running SQL queries
  • Staging tables for protecting the original raw dataset
  • Window functions for duplicate detection
  • ROW_NUMBER() and PARTITION BY for identifying duplicate records
  • String functions such as TRIM() for text standardisation
  • STR_TO_DATE() for date conversion
  • Self-joins for filling missing values where appropriate
  • Data quality checks for nulls, blanks and unusable records

Structured MySQL Cleaning Workflow

The project applies a clear MySQL cleaning process to transform a raw layoffs dataset into a cleaner and more analysis-ready version. The full query workflow is documented in the GitHub repository.

Cleaning Focus Areas

  • Creating staging tables to preserve the original raw data
  • Identifying duplicate records using window functions
  • Removing duplicate records from the cleaned staging table
  • Trimming and standardising company names
  • Standardising inconsistent industry values
  • Cleaning country values by removing unwanted punctuation
  • Converting date fields into a proper MySQL date format
  • Handling null and blank values in key fields
  • Using matching company records to populate missing industry values where possible
  • Removing records without usable layoff information
  • Dropping helper columns after the cleaning process was complete

Clean data is the foundation of accurate analysis. This project shows that I can take a raw dataset, protect the original data through staging tables and apply a logical MySQL workflow to improve consistency, completeness and usability.
